# CONTEXT

## Glossary

- **Shared skills repository** — the maintained project context for reusable Hermes skills and their supporting workflow assets.
- **Skill** — a reusable, documented workflow packaged as a directory containing `SKILL.md` and, when needed, supporting references, templates, scripts, or assets.
- **Adapted skill** — a local skill derived from one or more upstream skills and tracked with explicit provenance in frontmatter metadata.
- **Upstream skill** — a source skill from a vendored upstream repository under `upstream/`.
- **Provenance tracking** — the practice of recording exactly which upstream skill paths and pinned SHAs an adapted skill derives from.
- **Supporting workflow assets** — non-`SKILL.md` files that help a skill operate, such as references, templates, scripts, and assets.
- **Repository hygiene** — the conventions that keep the shared skills repository maintainable, diffable, and clean.
